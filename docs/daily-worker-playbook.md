# Daily Worker Playbook

This playbook turns Harness Engineering into daily behavior.

## 1. Start with intent

Before using AI, state:

- The goal
- The decision or output needed
- What context is known
- What constraints matter
- What role AI should play

Example:

> Help me prepare for this discussion. Track assumptions, unresolved questions, and decisions. Do not decide for the team.

## 2. Feed context as work unfolds

Do not wait until the end. Add context incrementally:

- Meeting notes
- Thread summaries
- Draft messages
- Query results
- Decisions
- Corrections from other people

The harness improves when it sees the flow of work, not just the final artifact.

## 3. Ask AI to challenge interpretation

Use AI to test reasoning:

- What are we assuming?
- What evidence supports this?
- What else could explain these results?
- What validation step would distinguish the cases?
- What would a skeptical reviewer ask?

## 4. Preserve human agency

AI can propose, summarize, and challenge. Humans still own:

- Priorities
- Risk tradeoffs
- Approvals
- Commitments
- Sensitive context decisions
- Final judgment

## 5. Capture learning

At the end of meaningful work, ask:

- What did we learn?
- What pattern should we reuse?
- What mistake should we prevent next time?
- What checklist or eval would catch this earlier?
- What should be turned into a template?

## 6. Make learning reusable

Reusable outputs can include:

- A checklist
- A prompt pattern
- A decision log
- A troubleshooting workflow
- A private eval
- A project issue
- A playbook update

The goal is not more documentation. The goal is operational memory that improves the next execution.
